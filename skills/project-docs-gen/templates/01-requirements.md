# 요구사항정의서 템플릿

이 템플릿을 기반으로 사용자 컨텍스트를 채워 요구사항정의서를 작성한다.
모든 후속 문서가 이 문서의 카테고리와 ID 체계를 따르므로 가장 먼저 작성해야 한다.

## 의존 문서

- 없음 (첫 번째 문서)

## 사용자에게 확인할 것

- 프로젝트 목표와 배경
- 주요 사용자(타겟)
- 핵심 기능 목록
- 비기능 요구사항 (성능, 보안 등)
- 프로젝트 일정/마일스톤
