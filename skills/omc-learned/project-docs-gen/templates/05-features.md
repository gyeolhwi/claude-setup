# 기능명세서 템플릿

## 의존 문서

- 요구사항정의서 (`requirements.md`) — 요구사항 ID, 카테고리
- 정보구조도 (`sitemap.md`) — 화면 ID, URL

## 사용자에게 확인할 것

- 각 기능의 상세 동작
- 입력/출력 데이터
- 예외 처리 방식
- 우선순위 변경사항

## 2. 기능 상세

### FEAT-AUTH-001: 로그인

**개요:** 이메일/비밀번호로 로그인

**화면:** SCR-AUTH-001 (`/login`)

**입력값:**
| 필드 | 타입 | 필수 | 검증 규칙 |
|------|------|------|---------|
| email | string | Y | 이메일 형식 |
| password | string | Y | 8자 이상 |

**처리 흐름:**
1. 사용자가 이메일/비밀번호 입력
2. 유효성 검사
3. 서버 인증 요청
4. 성공 시 토큰 저장 후 홈으로 이동
5. 실패 시 에러 메시지 표시

**성공 응답:** 홈 페이지로 리다이렉트

**예외 처리:**
| 상황 | 처리 방식 |
|------|---------|
| 이메일 미존재 | "이메일 또는 비밀번호가 올바르지 않습니다" |
| 비밀번호 불일치 | "이메일 또는 비밀번호가 올바르지 않습니다" |
| 5회 이상 실패 | 계정 일시 잠금 (30분) |

---
