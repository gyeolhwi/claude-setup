# 시나리오 케이스 템플릿

## 의존 문서

- 전체 문서 참조 (마지막에 작성)
- 기능명세서 (`features.md`) — 기능별 예외 처리
- 플로우차트 (`flowchart.md`) — 사용자 흐름
- 역할/권한 매트릭스 (`roles.md`) — 권한별 접근 테스트

## 사용자에게 확인할 것

- 테스트 우선순위 (핵심 기능 먼저)
- 엣지 케이스 범위
- 성능 테스트 포함 여부

## 2. 시나리오 케이스 상세

### 인증 (AUTH)

#### TC-AUTH-001: 정상 로그인

| 항목 | 내용 |
|------|------|
| 관련 기능 | FEAT-AUTH-001 |
| 사전 조건 | 가입된 계정 존재 |
| 테스트 데이터 | email: test@example.com, password: Test1234! |

**시나리오:**
1. 로그인 페이지 접속
2. 올바른 이메일/비밀번호 입력
3. 로그인 버튼 클릭

**예상 결과:**
- 홈 페이지로 이동
- 헤더에 사용자 이름 표시
- 로컬스토리지에 accessToken 저장

**상태:** `미실행` / `통과` / `실패`

---

#### TC-AUTH-002: 잘못된 비밀번호 로그인

| 항목 | 내용 |
|------|------|
| 관련 기능 | FEAT-AUTH-001 |
| 사전 조건 | 가입된 계정 존재 |
| 테스트 데이터 | email: test@example.com, password: WrongPassword |

**시나리오:**
1. 로그인 페이지 접속
2. 올바른 이메일 + 잘못된 비밀번호 입력
3. 로그인 버튼 클릭

**예상 결과:**
- 에러 메시지: "이메일 또는 비밀번호가 올바르지 않습니다"
- 페이지 이동 없음
- 비밀번호 필드 초기화

**상태:** `미실행`

---

#### TC-AUTH-003: 5회 로그인 실패 후 계정 잠금

| 항목 | 내용 |
|------|------|
| 관련 기능 | FEAT-AUTH-001 |
| 사전 조건 | 가입된 계정 존재 |

**시나리오:**
1. 잘못된 비밀번호로 5회 로그인 시도

**예상 결과:**
- 6번째 시도 시 계정 잠금 메시지
- 30분 후 자동 해제

**상태:** `미실행`
